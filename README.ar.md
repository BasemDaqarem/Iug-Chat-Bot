# IUG Chatbot — شات بوت الجامعة الإسلامية بغزة

**English: [README.md](README.md)**

مساعد جامعي عربي يعتمد RAG (استرجاع + توليد): يقرأ معرفة الجامعة من MongoDB،
يبحث فيها بحثاً هجيناً (دلالي Jina + لفظي BM25)، ويجيب عبر LLM متوافق مع OpenAI
(OpenRouter حالياً) — مع مصادقة JWT، وأدوار (زائر/طالب/موظف/أدمن) تعزل البيانات
بنيوياً قبل أن تصل للنموذج، ولوحة إدارة كاملة للمحتوى والصلاحيات.

**الحالة**: 311 اختباراً آلياً ✅ · تقييم ذهبي 6/0 ✅ · اختبار حيّ 99 سؤالاً ·
مراجعة أمنية (9 بنود) مُعالجة · جاهز للنشر على Render.

---

## أبرز القدرات

| القدرة | كيف |
|--------|-----|
| **إجابات من معرفة الجامعة فقط** | استرجاع هجين (دلالي + BM25 عبر RRF) ثم توليد مقيّد بالمقاطع |
| **أدوار وصلاحيات** | زائر/طالب/موظف/أدمن — فلترة الملفات **قبل** البحث؛ ملف «موظفون داخلي» لا يدخل ترتيب نتائج طالب أصلاً |
| **ربط بيانات الطالب** | «رئيس قسمي؟ معدلي؟ التدريب الميداني؟» تُجاب من ملفه وتخصّصه تلقائياً |
| **ذاكرة محادثة دلالية** | آخر 5 أدوار بمتجهات محفوظة؛ يُحقن ذو الصلة فقط؛ «وللماجستير؟» تفهم سابقها |
| **فهم العامية** | «كيف اجل الفصل» تطابق «تأجيل الدراسة» في البيانات |
| **الأحدث يفوز** | عند تضارب معلومة بين ملفين تُعتمد نسخة الملف الأحدث تحديثاً |
| **كبح الهلوسة** | التفاصيل الإجرائية من المقاطع فقط؛ أسئلة الأشخاص/السياسة خارج النطاق تُرفض بأدب |
| **لوحة إدارة** | رفع JSON بنشر فوري، تصنيفات وأدوار لكل ملف، حذف، نسخ وإرجاع، سجل تدقيق |
| **صلابة** | كاش آمن للخصوصية، حدود معدل، أخطاء موحّدة، فهارس فريدة، أسرار fail-closed |

## التشغيل محلياً

```bash
pip install -r requirements.txt
cp .env.example .env      # عبّئ المفاتيح (انظر جدول البيئة أدناه)
python server.py          # ← http://127.0.0.1:8000/app (الواجهة) و /docs (التوثيق)
```

- واجهة الطلاب: `http://127.0.0.1:8000/app/`
- لوحة الإدارة: `http://127.0.0.1:8000/app/admin.html` (تتطلب حساباً بدور admin)
- بوابة الموظف: تسجيل دخول برقم وظيفي يوجّه لواجهة الموظف

### الاختبارات والتقييم

```bash
PYTHONIOENCODING=utf-8 python -m pytest tests/ -q     # 311 اختباراً
PYTHONIOENCODING=utf-8 python eval/run_eval.py        # تقييم ذهبي حيّ (يحتاج .env)
```

> `PYTHONIOENCODING` مطلوب على Windows بسبب الإيموجي في المخرجات.

## متغيرات البيئة (.env)

| المفتاح | الوصف | ملاحظات |
|---|---|---|

| `CHAT_API_URL` / `CHAT_API_KEY` / `CHAT_API_MODEL` | مزوّد المحادثة (متوافق OpenAI) | OpenRouter + `openai/gpt-oss-20b` |
| `EMBED_API_URL` / `EMBED_API_KEY` / `EMBED_MODEL` | مزوّد التضمين | Jina `jina-embeddings-v3` |
| `JWT_SECRET` | توقيع التوكنات | **سرّي، ≥32 حرفاً** — الإنتاج يرفض الإقلاع بسرّ ضعيف |
| `ADMIN_API_KEY` | مفتاح مسارات الإدارة القديمة | **سرّي** |
| `ADMIN_BOOTSTRAP_ID` / `_PASSWORD` / `_NAME` | أول حساب أدمن (يُنشأ عند الإقلاع إن لم يوجد) | احذفهما بعد أول دخول |
| `API_ENV` | `development` / `production` | production تخفي `/docs` وتشدّد الأمان |
| `INDEX_BACKEND` | `disk` / `mongo` | mongo للنشر (قرص Render مؤقت) |
| `SESSION_BACKEND` | `mongo` / `memory` | mongo افتراضياً |
| `ALLOW_PUBLIC_REGISTRATION` | `true` تجريبياً | `false` عند الربط بنظام القبول |
| `LEGACY_UNCATALOGUED_FILES_PUBLIC` | إظهار الملفات السابقة لنظام السجل | `true` للتجربة؛ `false` بعد تصنيفها من اللوحة |
| `RATE_LIMIT_CHAT_PER_MIN` / `RATE_LIMIT_LOGIN_PER_MIN` | حدود المعدل | 30 / 10 |
| `MEMORY_MIN_SIM` | عتبة صلة ذاكرة المحادثة | 0.45 |
| `API_CORS_ORIGINS` | أصول CORS | فارغة عند تقديم الواجهة من نفس الخادم |

## النشر على Render

الملف [`render.yaml`](render.yaml) جاهز (Blueprint). الخطوات كاملة:

1. **ارفع المستودع إلى GitHub** (بدون `.env` — مُتجاهَل أصلاً في `.gitignore`).
2. في [MongoDB Atlas](https://cloud.mongodb.com): **Network Access ➜ Add IP ➜ `0.0.0.0/0`**
   (خطة Render المجانية بلا IP ثابت — الحماية ببقية الطبقات: مستخدم Mongo بكلمة قوية + TLS).
3. في [Render](https://dashboard.render.com): **New ➜ Blueprint ➜** اختر المستودع —
   سيقرأ `render.yaml` وينشئ خدمة `iug-chatbot`.
4. **Environment**: عبّئ الأسرار المعلَّمة `sync: false`
   (`MONGO_URI`، مفاتيح المحادثة والتضمين، `ADMIN_BOOTSTRAP_ID/PASSWORD`).
   `JWT_SECRET` و`ADMIN_API_KEY` يولّدهما Render تلقائياً.
5. **Deploy** وانتظر البناء (~2-4 دقائق). أول إقلاع يبني/يحمّل فهرس المتجهات من Mongo.
6. تحقق: `https://<الخدمة>.onrender.com/health` ← ‏`status: ok`، ثم جرّب `/app`.
7. ادخل بحساب الأدمن (قيم البوتستراب) ➜ **احذف `ADMIN_BOOTSTRAP_PASSWORD` من اللوحة**.

**ملاحظات الخطة المجانية**: تنام الخدمة بعد ~15 دقيقة خمول (أول طلب بعد النوم
يتأخر ~30-60 ثانية) · 750 ساعة/شهر · القرص مؤقت — لذلك الفهرس والجلسات في Mongo.
كل `git push` إلى `main` يعيد النشر تلقائياً.

## نقاط الـ API الرئيسية

| النقطة | الوصف |
|---|---|
| `GET /health` | جاهزية الخدمة وإحصاءاتها |
| `POST /api/auth/login` · `/register` · `GET /api/auth/me` | مصادقة JWT (bcrypt) |
| `POST /api/chat` | محادثة الدور الموثّق (الهوية والدور من التوكن) |
| `POST /api/chat/guest` | محادثة الزائر (الملفات العامة فقط، حدّ بالـ IP) |
| `POST /api/chat/student` | محادثة الطالب مع دمج ملفه الأكاديمي |
| `GET/DELETE /api/sessions/me/history` | سجل محادثة صاحب التوكن فقط |
| `GET /api/admin/files` · `POST` · `PATCH …/access` · `DELETE` | إدارة الملفات (أدمن) |
| `POST /api/admin/files/{id}/process` · `/publish` · `/rollback/{v}` | دورة النشر والنسخ |
| `GET/POST/PATCH /api/admin/employees` | حسابات الموظفين |
| `GET /api/admin/audit` | سجل التدقيق |
| `GET /api/cache/stats` · `POST /api/cache/clear` | مراقبة الكاش (أدمن) |

كل الأخطاء بصيغة موحّدة:
`{"success": false, "error": {code, message, details, timestamp, path}}`
برموز حالة صحيحة (401/403/404/409/422/429/502/503) — لا 200 عند الفشل أبداً.

## رحلة السؤال في المحرك

```
سؤال الطالب
 ├─ حراسة الخصوصية (سؤال عن طالب آخر؟ → رفض فوري بلا LLM)
 ├─ بناء استعلام البحث (نصّي بحت — صفر LLM إضافي):
 │    قسمي/الكلية/التدريب ← + تخصص الطالب · سؤال مبهم ← يرث سابقه (بسلسلة)
 │    عامية ← + المصطلح الرسمي (اجل ← تأجيل الدراسة)
 ├─ فلترة الصلاحية: ملفات دور السائل فقط تدخل الترشيح (عزل بنيوي)
 ├─ استرجاع هجين (دلالي + BM25 عبر RRF) + مذكرة «الأحدث يفوز» عند تعدد المصادر
 ├─ ذاكرة دلالية: أدوار سابقة ذات صلة فقط (متجهات محفوظة، fallback آمن)
 └─ LLM: سؤال حرفي + ملف الطالب + مقاطع + ذاكرة — بقواعد كبح الهلوسة
```

```
app/
├── config.py            ← كل الإعدادات (المكان الوحيد لقراءة .env)
├── prompts.py           ← قوالب البرومت العربية (نصها = سلوك البوت)
├── db.py                ← اتصال MongoDB موحّد (قاعدتان منطقيتان)
├── chunking.py          ← وثائق → مقاطع (+ ترويسة [ملف: الاسم])
├── text_norm.py / lexical.py / retrieval.py ← تطبيع عربي + BM25 + دمج RRF
├── embeddings.py / index_store.py ← عميل Jina + حفظ الفهرس (disk/mongo)
├── query_rewrite.py     ← توسيع الاستعلام: ذاتي/سياقي/مصطلحات رسمية
├── sessions.py          ← ذاكرة المحادثة الدلالية (5 أدوار FIFO بمتجهات)
├── cache.py / ratelimit.py ← كاش TTL+LRU آمن + حدود معدل
├── llm.py               ← عميل المحادثة (retry + أخطاء واضحة باسم المزوّد)
├── auth.py / tokens.py / rbac.py ← bcrypt + JWT + الأدوار
├── privacy.py / context_builder.py ← حراسة الخصوصية + السياق الخاص بالدور
├── file_catalog.py      ← سجل الملفات: تصنيفات/أدوار/مالك/نسخ/تبنّي القديم
├── admissions.py / audit.py ← إجابات قبول منظّمة + سجل تدقيق
├── knowledge_base.py / uploaded_files.py / chatbot.py ← التنسيق العام
└── api/                 ← FastAPI: schemas/deps/middleware/errors/routers
frontend/                ← واجهات HTML/CSS/JS خفيفة RTL:
                           index (دخول/تسجيل) · chat · admin (لوحة) · employee
eval/run_eval.py         ← تقييم ذهبي (شغّله بعد أي تعديل على البرومت)
tests/                   ← 311 اختباراً (أمن، أدوار، ذاكرة، استرجاع، API…)
docs/                    ← تقارير التحسينات والاختبارات
```

## قواعد للمطوّرين

- `config.py` وحده يقرأ `.env` — لا `os.getenv` في مكان آخر.
- نص `prompts.py` سلوك وليس تفصيلاً: أي تعديل ⇒ شغّل `eval/run_eval.py` وقارن.
- تعديل `chunking.py` يستلزم إعادة فهرسة (بصمة الفهرس تتكفل بذلك تلقائياً).
- بيانات الطلاب/الموظفين الشخصية لا تُفهرَس في RAG أبداً
  (`_ALWAYS_EXCLUDE_FROM_RAG`) — تصل النموذج فقط كسياق خاص لصاحبها الموثّق.
- حسابات التجربة على القاعدة الحيّة: أنشئها ثم **احذفها** (auth + جلستها).
