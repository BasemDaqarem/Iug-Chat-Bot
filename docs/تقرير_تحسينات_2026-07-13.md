## 2) ربط أسئلة الطالب الشخصية ببياناته + فهم أسئلة المتابعة
`757fd2d` — إصلاح جذري في طبقة الاسترجاع (ملف جديد: `app/query_rewrite.py`)

**المشكلة (شكوى باسم الأصلية):**
- «كيف اتواصل مع رئيس **قسمي**؟» → أجاب عن قسم التمريض بدل قسمه
- «كيف بدي اجل الفصل» ثم «كم هيكلفني رسوم **هذا الطلب**» → أجاب رسوم التخرج

**التشخيص:** محرك البحث كان يرى **نص السؤال الحرفي فقط**:
- «قسمي» لا تحمل اسم التخصص → البحث لا يعرف أي قسم يقصد
- «هذا الطلب» لا تعني شيئاً للبحث → جلب مقاطع موضوع آخر
- (سجل المحادثة كان يصل للنموذج أصلاً — المشكلة في المقاطع المسترجعة لا الذاكرة)

**الحل — عمليات نصية بحتة بصفر استدعاءات ذكاء اصطناعي إضافية:**
- توسيع ذاتي: «قسمي/تخصصي/كليتي…» → يُلحق تخصص الطالب (من ملفه) باستعلام البحث
- وراثة السياق: سؤال متابعة (إشارة عائدة أو سؤال قصير) → يرث نص السؤال السابق
- **البحث يرى الاستعلام الموسّع؛ النموذج والسجل والكاش يرون سؤال الطالب الحرفي**

**النتيجة:** «رئيس قسمي» → إيميل رئيس قسم هندسة الحاسوب من أول مرة؛
«كم هيكلفني رسوم هذا الطلب» → 10 دنانير رسوم التأجيل.

## 4) «التدريب الميداني» يُجاب من كلية أخرى وكأنه لكلية الطالب
`f11a8bc` — تشخيص أسئلة الكلية الضمنية + الصدق عند غياب البيانات

**المشكلة:** «كيف ممكن انجز التدريب الميداني» أجاب بإجراءات كلية تكنولوجيا
المعلومات وعرضها كأنها إجراءات كلية الطالب (الهندسة).

**اكتشاف جوهري:** فُتّشت قاعدة البيانات كاملة — **لا توجد أي معلومات تدريب
ميداني لكلية الهندسة أصلاً**. المقطع الوحيد يخص تكنولوجيا المعلومات.
لا كود يستخرج معلومة غير موجودة — هذه فجوة بيانات.

**الحل (طبقتان):**
1. مواضيع مرتبطة بكلية الطالب ضمنياً (التدريب الميداني/مشروع التخرج/الخطة
   الدراسية) تُشخصن بتخصصه في البحث تلقائياً — حتى بلا كلمة «قسمي»؛ وإذا سمّى
   كلية أخرى صراحة لا يُفرض تخصصه عليها
2. قاعدة برومت ملزمة: عند غياب بيانات كلية الطالب يبدأ الجواب بتوضيح ذلك
   («لا تتوفر لدي تفاصيل التدريب الميداني لكليتك…») ثم يعرض المتوفر منسوباً
   لكليته الأصلية كمرجع عام — لا كإجراءات كليته

**⬅️ مطلوب من باسم:** إضافة بيانات التدريب الميداني لكلية الهندسة (بنفس بنية
مقطع تكنولوجيا المعلومات). لحظة رفعها يجيب البوت تلقائياً بلا أي تعديل كود.

## 5) ذاكرة محادثة قصيرة دلالية (5 أدوار بمتجهات)
`c6a8705` — تنفيذ مواصفة الذاكرة المطلوبة

**المطلوب:** آخر 5 أدوار فقط (FIFO) كلٌّ بسؤاله وجوابه ومتجهه المحسوب مرة
واحدة؛ قبل كل سؤال جديد تُقارن المتجهات ويُحقن **ذو الصلة فقط** في البرومت
مع تعليمة ملزمة؛ بلا طبقات جديدة وبلا كسر أي شيء.

**التنفيذ (3 ملفات فقط):**
- `config.py`: حد الذاكرة 5 + عتبة الصلة `MEMORY_MIN_SIM=0.45`
- `sessions.py`: تخزين متجه السؤال مع الدور (مرة واحدة — لا يُعاد حسابه)؛
  انتقاء `relevant_turns` بتشابه cosine (الدور الأحدث يُحقن دائماً لاستمرارية
  الحوار)؛ التعليمة الملزمة قبل بيانات الذاكرة
- `chatbot.py`: التضمين من `embed_query` المُكاش (مجاني — الاسترجاع ضمّن نفس
  السؤال قبل لحظات)؛ **فشل التضمين → عودة آمنة للسلوك القديم والشات يستمر**

**ضمانات:** عقد API لم يتغير (المتجهات لا تظهر في الاستجابات — تحقق حي)؛
FIFO عبر آلية `$slice` الموجودة في Mongo أصلاً؛ السرعة كما هي.

اجل/أجل/بأجل → «تأجيل الدراسة» · اسجل/بسجل → «تسجيل المساقات» · انسحب → «انسحاب»

## 8) «رئيس القسم» بأل التعريف لا يعرف قسم الطالب
`c16e937` — الصيغ المعرفة المجردة = ملكية ضمنية

**المشكلة (شكوى باسم الأخيرة):** «كيف ممكن اتواصل مع رئيس **القسم**؟» أجاب
«لا توجد معلومات» — بينما «رئيس **قسمي**» كانت تجيب إيميل أحمد مهدي.

**التشخيص:** محفزات التوسيع كانت ياء الملكية فقط (قسمي/كليتي). «القسم» بأل
التعريف لم تُفعّلها، فلم يصل مقطع رئيس قسم الحاسوب لأفضل نتائج البحث.
(البوت عرف قسمه من ملفه وأجاب بصدق إن المقطع غير موجود عنده.)

**الحل:** «القسم/الكلية/قسمنا/كليتنا» المجرّدة من طالب مسجّل = قسمه هو —
أضيفت للمحفزات. الذكاء اللغوي: تسمية قسم آخر («قسم التمريض») تأتي بلا أل
تعريف على الكلمة المحفّزة، فلا تتأثر.

**النتيجة:** «رئيس القسم؟» → amahdi@iugaza.edu.ps ✅ ·
«مين رئيس قسم التمريض؟» → رئيسة التمريض (بلا تلوث بتخصصه) ✅

## المعمارية بعد التحسينات — رحلة السؤال

```
سؤال الطالب
   │
   ├─ حراسة الخصوصية (سؤال عن طالب آخر؟ → رفض فوري)
   │
   ├─ بناء استعلام البحث (query_rewrite — نصي بحت، صفر LLM):
   │     1. توسيع ذاتي: قسمي/القسم/التدريب الميداني… + تخصص الطالب
   │     2. وراثة السياق: سؤال مبهم يرث سابقه (+ قفزة سلسلة)
   │     3. مصطلح قانوني: اجل → تأجيل الدراسة
   │
   ├─ استرجاع هجين (دلالي + BM25 عبر RRF) بالاستعلام الموسّع
   │
   ├─ ذاكرة دلالية: متجه السؤال يُقارن بمتجهات آخر 5 أدوار المحفوظة
   │     → يُحقن ذو الصلة فقط + الأحدث دائماً (فشل التضمين → fallback آمن)
   │
   └─ LLM: يرى (سؤال الطالب الحرفي + ملفه الخاص + المقاطع + الذاكرة المنتقاة)
         قواعد: لا اختلاق تفاصيل إجرائية · نسب المعلومة لكليتها الأصلية
         · المبهم يُفهم من السجل
```
